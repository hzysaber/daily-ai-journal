## 📝 技术笔记：Prompt Engineering 高级技巧：Few-shot、CoT、Self-Consistency

在大语言模型（LLM）的应用中，Prompt Engineering是连接人类意图与模型能力的核心桥梁。随着应用场景的复杂化，简单的指令往往无法满足需求。本文将深入探讨三种高级提示技巧——Few-shot、思维链（Chain-of-Thought, CoT）和自洽性（Self-Consistency），它们共同构成了提升模型推理能力与输出可靠性的关键工具箱。

### 1. Few-shot（少样本学习）

**核心概念**：Few-shot 是一种在提示中直接提供少量“输入-输出”示例的技术。它通过“情境学习”，让模型在推理时遵循示例所隐含的模式、格式或逻辑，从而无需重新训练即可适应新任务。它介于 Zero-shot（无示例）和微调之间。

**技术原理**：模型在大规模预训练中学会了强大的模式匹配和泛化能力。Few-shot 提示相当于在上下文窗口中激活了与任务最相关的特定能力。示例的质量、数量和顺序对结果有显著影响。

**代码示例（伪代码）**：
```python
prompt = f"""
任务：将用户评论的情感分类为“正面”或“负面”。
用户评论：这家餐厅的食物太美味了，服务也很周到。
情感：正面

用户评论：等了半小时才上菜，而且菜品是冷的。
情感：负面

用户评论：{user_input}
情感：
"""
# 调用LLM API获取分类结果
response = llm.generate(prompt)
```

**实际应用场景**：
*   **格式对齐**：要求模型以特定JSON、Markdown表格或自定义模板输出结果。
*   **风格模仿**：生成符合特定作者或文档风格的文本。
*   **文本分类**：如情感分析、意图识别，通过示例明确定义分类边界。

**常见坑点**：
1.  **示例质量不高**：包含错误或模糊的示例会导致模型学到错误模式。
2.  **示例数量失衡**：在分类任务中，各类别的示例数量应尽量均衡，否则模型会偏向多数类。
3.  **示例与任务不相关**：提供无关的示例会“污染”上下文，干扰模型判断。

### 2. Chain-of-Thought (CoT，思维链)

**核心概念**：CoT 鼓励模型在给出最终答案前，先生成一个逐步的推理过程。它将一个复杂问题分解为一系列更小、更易处理的中间步骤。

**技术原理**：LLM 本质是下一个 token 的预测器。通过要求它输出中间推理步骤，相当于强制模型在预测“最终答案”这个 token 之前，先生成所有中间推理 token。这引导模型的内部注意力机制分配到逻辑推理上，而非直接模式匹配，从而显著提升在算术、逻辑和常识推理任务上的表现。

**代码示例（伪代码）**：
```python
# Zero-shot CoT: 通过一个神奇咒语触发
prompt_cot = f"""
问题：一个文具店卖笔记本，每本5元。小明买了3本，付给老板50元，应该找回多少钱？
让我们一步步思考。
"""

# Few-shot CoT: 提供一个带推理过程的示例
prompt_few_shot_cot = f"""
问题：农场有12只鸡，卖掉5只后，又孵化了8只小鸡，现在农场有多少只鸡？
思考：最初有12只鸡。卖掉5只后剩下 12 - 5 = 7只。然后又增加了8只，所以总共是 7 + 8 = 15只。
答案：15

问题：{user_question}
思考：
"""
response = llm.generate(prompt_cot) # 模型会输出完整的推理过程
```

**实际应用场景**：
*   **复杂数学应用题**：特别是多步计算。
*   **逻辑谜题与推理**：如逻辑判断、序列找规律。
*   **代码调试与生成**：让模型解释每一步的代码逻辑。
*   **需要解释或可验证的决策**：例如医疗诊断建议（辅助）、法律条款分析。

**常见坑点**：
1.  **过度推理**：对于简单问题，强行要求CoT可能使模型“画蛇添足”，引入错误。
2.  **误导性推理链**：模型可能生成一个看似合理但结论错误的推理过程（“一本正经地胡说八道”）。
3.  **资源消耗**：输出更长，会增加延迟和成本。

### 3. Self-Consistency（自洽性）

**核心概念**：Self-Consistency 是对 CoT 的增强。它不是只采样一次推理路径，而是对同一个问题进行多次 CoT 采样（通常使用较高的 temperature），然后从所有采样路径的结果中，选择出现频率最高的答案作为最终输出。

**技术原理**：其哲学是“多数人的智慧”。对于一个有确定答案的问题，正确的推理路径虽然可能不止一条，但错误的路径则多种多样。通过多次采样，正确答案在所有路径结果中出现的概率会更高。这相当于一个无需训练的集成学习方法，提升了结果的鲁棒性。

**伪代码示例**：
```python
import collections

prompt_template = """
问题：{question}
让我们一步步思考。
"""
question = "一个数除以8余3，除以5余2，这个数最小是多少？"
answers = []

# 进行多次CoT采样
for i in range(5): # 采样5次
    response = llm.generate(
        prompt_template.format(question=question),
        temperature=0.7 # 较高的温度以产生多样性
    )
    # 从response中解析出最终答案（数字部分）
    final_answer = parse_answer_from_cot(response)
    answers.append(final_answer)

# 多数投票
most_common_answer = collections.Counter(answers).most_common(1)[0][0]
print(f"自洽性结果：{most_common_answer}")
```

**实际应用场景**：
*   **高风险决策**：需要极高置信度答案的场景，如科学计算、数据验证。
*   **开放域问答**：特别是事实性问题，能有效减少随机错误。
*   **提升CoT的稳定性**：当模型有时能做对有时不能做对时，Self-Consistency能提高“做对”的概率。

**常见坑点**：
1.  **计算成本高昂**：需要多次调用模型，时间和金钱成本成倍增加。
2.  **对开放式、主观性问题无效**：对于没有唯一标准答案的问题（如创意写作），“多数投票”没有意义。
3.  **可能放大的偏见**：如果模型对某类错误有系统性偏好，多次采样可能无法纠正，反而会强化该错误。

**总结**：Few-shot 提供模式，CoT 提供深度推理能力，Self-Consistency 提供结果的可靠性保障。在实际应用中，它们常组合使用：例如，用 **Few-shot CoT** 示例来引导模型进行分步推理，并应用 **Self-Consistency** 策略来筛选出最稳健的答案。理解每种技巧的原理与局限，根据具体任务（分类、推理、生成）和需求（速度、成本、准确率）进行选择和调优，是构建高效、可靠 LLM 应用的关键。

## 💻 代码片段：用Python实现一个简单的token计数器

一个简单的 Token 计数器，基于常见语言规则（单词、标点）进行估算。适用于大多数英文文本处理场景，如 API 调用前的长度预估。

```python
import re
from typing import Dict

class TokenCounter:
    """
    基于规则的简单 Token 计数器。

    该计数器通过正则表达式将文本分割为单词和标点符号，
    并按照常见语言模型（如 GPT）的规则估算 token 数量。
    注意：这是一个近似值，实际 token 数可能因分词器不同而略有差异。

    Attributes:
        pattern (re.Pattern): 用于分割文本的正则表达式模式。
    """

    def __init__(self) -> None:
        # 匹配单词、数字、标点符号及空格
        self.pattern = re.compile(r"\w+|[^\w\s]|\s+")

    def count_tokens(self, text: str) -> int:
        """
        计算给定文本的估算 token 数量。

        Args:
            text: 需要计数的输入文本。

        Returns:
            int: 估算的 token 数量。
        """
        if not text:
            return 0
        
        # 分割文本为 token
        tokens = self.pattern.findall(text)
        
        # 统计逻辑：单词（连续字母/数字）通常为 1 token，标点符号和空格通常也为 1 token
        return len(tokens)


def main():
    """展示 TokenCounter 的使用示例。"""
    counter = TokenCounter()
    
    # 示例文本
    examples = [
        "Hello, world!",
        "This is a simple test.",
        "Token counting is useful for API pricing.",
        "你好，世界！",
        ""  # 空字符串测试
    ]
    
    for text in examples:
        token_count = counter.count_tokens(text)
        print(f"文本: '{text}' -> Token数: {token_count}")


if __name__ == "__main__":
    main()
```

示例运行结果：
```
文本: 'Hello, world!' -> Token数: 5
文本: 'This is a simple test.' -> Token数: 6
文本: 'Token counting is useful for API pricing.' -> Token数: 8
文本: '你好，世界！' -> Token数: 3
文本: '' -> Token数: 0
```
