## 📝 技术笔记：Function Calling 与 Tool Use 的实现原理

Function Calling（函数调用）与 Tool Use（工具使用）是大语言模型（LLM）扩展能力的关键技术，使模型能够与外部工具、API 或数据库进行交互，从而突破纯文本生成的局限。本文将从概念、原理到实践，系统梳理其实现机制。

### 核心概念

**Function Calling** 指模型根据用户输入，自主决定调用预定义的函数（如 API、计算工具），并生成结构化的参数（通常为 JSON）的过程。**Tool Use** 是更广义的概念，强调模型将外部工具（包括函数、服务、硬件）作为能力延伸。两者本质相同：模型不再只生成文本，而是输出可执行的指令。

传统LLM是封闭系统，而通过 Function Calling，模型可成为“协调者”——解析意图、生成调用指令，并处理返回结果，最终整合成连贯回答。

### 技术原理

实现分三个核心步骤：

1. **函数声明与描述**  
   开发者需在调用模型时，以结构化方式（如JSON Schema）声明可用的函数列表，包括函数名、功能描述、参数名称、类型和约束。这相当于给模型一份“工具说明书”。

2. **意图解析与调用生成**  
   模型接收用户输入（如“北京今天天气如何”），通过内部推理匹配合适的函数，并生成调用指令。指令包含函数名和符合约束的参数（如 `get_weather(city="北京")`）。模型在此阶段充当“路由器”。

3. **执行与结果整合**  
   外部系统执行函数调用，将结果（如天气数据）返回给模型。模型再将结果转化为自然语言回答用户。

整个流程通常通过同步或异步API实现，开发者需管理函数执行和结果回传。

### 代码示例

以下为简化伪代码，展示与 OpenAI API 风格的集成：

### 实际应用场景

1. **实时信息查询**：天气、新闻、股票等动态数据获取。
2. **数据库操作**：自然语言转 SQL 查询（Text-to-SQL），实现数据交互。
3. **系统控制**：管理智能家居、执行运维命令（如重启服务）。
4. **代码执行**：模型生成代码片段，由沙箱环境执行并返回结果。
5. **多工具链式调用**：复杂任务拆解为多步函数调用（如先搜索再计算）。

### 常见坑点

1. **函数描述不清晰**：  
   模型对模糊或歧义的描述（如“查询数据”）可能错误调用。解决方案：提供详细示例和明确约束（如“仅支持中文城市名”）。

2. **参数生成错误**：  
   模型可能生成无效值（如日期格式错误）。应对：在参数定义中加入 `enum` 枚举或正则表达式校验，并在代码层增加验证。

3. **错误处理缺失**：  
   函数执行可能失败（如网络超时）。需设计重试机制，并将错误信息返回模型，引导其生成友好提示或备选方案。

4. **安全风险**：  
   模型可能被诱导调用危险函数（如删除数据）。必须遵循最小权限原则，对敏感函数实施人工确认或权限校验。

5. **延迟与成本**：  
   多步调用增加延迟和 token 消耗。优化：合理设计函数粒度，缓存常见结果，使用并行调用减少轮次。

6. **多函数选择冲突**：  
   当多个函数功能重叠时，模型可能选择不当。建议优化函数命名和描述，或设置函数优先级权重。

Function Calling 标志着 LLM 从“思考者”向“行动者”的转变。随着模型推理能力的提升和工具生态的丰富，这一技术正推动 AI 应用走向更广阔的领域。开发者需在灵活性、安全性与可靠性间寻找平衡，方能构建稳健的智能系统。

# Python 实现简单 Token 计数器

以下是一个简单的 token 计数器，适用于基础的中文和英文文本分词。使用正则表达式实现简单的中文字符分割和英文单词识别。

```python
import re
from typing import List, Tuple


def count_tokens(text: str) -> Tuple[List[str], int]:
    """
    简单的 token 计数器，支持中英文混合文本
    
    Args:
        text: 输入的文本字符串
        
    Returns:
        Tuple[List[str], int]: (token 列表, token 数量)
        
    Examples:
        >>> tokens, count = count_tokens("你好，Hello world!")
        >>> print(tokens)
        ['你', '好', '，', 'Hello', 'world', '!']
        >>> print(count)
        6
    """
    if not text:
        return [], 0
    
    # 简单的分割规则：
    # 1. 中文字符单独作为 token
    # 2. 英文单词、数字作为 token
    # 3. 标点符号单独作为 token
    pattern = r"[\u4e00-\u9fff]|[a-zA-Z0-9]+|[^\w\s]"
    
    tokens = re.findall(pattern, text)
    return tokens, len(tokens)


# 使用示例
if __name__ == "__main__":
    # 测试文本
    sample_text = "Hello world! 你好，世界！Python 3.10 是一个好版本。"
    
    # 计算 token
    tokens, total_count = count_tokens(sample_text)
    
    print(f"原始文本: {sample_text}")
    print(f"Token 列表: {tokens}")
    print(f"Token 数量: {total_count}")
    print(f"平均每个 token 字符数: {len(sample_text) / total_count:.2f}")
    
    # 空字符串测试
    empty_tokens, empty_count = count_tokens("")
    print(f"\n空字符串测试: {empty_count} tokens")
```

## 说明
这是一个轻量级的 token 计数器，适用于基础的中英文混合文本处理。通过正则表达式将文本分割为中文字符、英文单词和标点符号三类 token。
