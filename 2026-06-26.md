## 📝 技术笔记：RAG（检索增强生成）的工程实践与优化技巧

## 核心概念
RAG（Retrieval-Augmented Generation）是一种将信息检索与大语言模型生成能力相结合的架构范式。它通过**检索外部知识库**为模型提供实时、精准的上下文，再基于这些上下文生成回答，从而有效缓解大模型的**幻觉问题**和**知识时效性**问题。其核心思想是：**不依赖模型内部记忆，而是动态“开卷考试”**。

一个经典的RAG流程包含四个关键阶段：
1.  **知识库构建**：将非结构化文档（如PDF、网页）进行分块、向量化，并存入向量数据库。
2.  **检索**：将用户查询向量化，在向量库中进行相似度搜索，获取最相关的文本块。
3.  **增强**：将检索到的相关文本块与原始查询组合，构造一个信息更丰富的提示。
4.  **生成**：将增强后的提示输入大语言模型，生成最终答案。

## 技术原理
RAG的技术实现依赖于**嵌入模型**和**向量检索**。

- **嵌入（Embedding）**：使用如`text-embedding-ada-002`、`bge-large-zh`等模型，将文本转换为高维空间的稠密向量（例如1536维）。语义相近的文本，其向量距离更近。
- **检索（Retrieval）**：在向量数据库（如FAISS、Milvus、Pinecone）中，通过计算查询向量与文档向量的余弦相似度或内积，返回Top-K个最相关的结果。
- **生成（Generation）**：采用提示工程，将检索结果格式化为结构化的上下文，约束模型基于给定事实生成回答，避免自由发挥。

**代码示例（基于Python的LangChain框架）**：
```python
from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import FAISS
from langchain.chains import RetrievalQA
from langchain.llms import OpenAI

# 1. 加载与分块
loader = TextLoader("knowledge.txt")
documents = loader.load()
text_splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
docs = text_splitter.split_documents(documents)

# 2. 向量化与存储
embeddings = OpenAIEmbeddings()
vectorstore = FAISS.from_documents(docs, embeddings)

# 3. 构建RAG链
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
qa_chain = RetrievalQA.from_chain_type(
    llm=OpenAI(temperature=0),
    chain_type="stuff",
    retriever=retriever,
    return_source_documents=True
)

# 4. 查询
query = "RAG的主要优势是什么？"
result = qa_chain({"query": query})
print(result["result"])
# 输出将基于检索到的文档内容生成
```

## 实际应用场景
1.  **企业知识库问答**：将内部文档、产品手册、政策规范构建为RAG系统，实现精准的员工自助查询。
2.  **智能客服**：结合产品FAQ和最新公告，为客服机器人提供实时、准确的知识支持。
3.  **学术研究助手**：在大量论文库中检索相关信息，辅助研究人员进行文献综述和观点验证。
4.  **法律与金融分析**：基于法规、合同或财报数据库，提供有依据的专业分析与摘要。

## 常见坑点与优化技巧
1.  **分块策略不当**：
    - **坑点**：分块过小丢失上下文，过大则引入噪声且成本高。
    - **优化**：根据文档类型调整`chunk_size`（通常500-1500字），并设置合理的`chunk_overlap`（50-200字）以保证语义连贯。对结构化文档（如报告）可使用基于标题的分块。

2.  **向量模型选择失误**：
    - **坑点**：通用嵌入模型在专业领域表现不佳。
    - **优化**：在特定领域（如医疗、法律）考虑微调嵌入模型，或选用在该领域预训练的模型（如`bge-large-zh`适合中文场景）。

3.  **检索召回率不足**：
    - **坑点**：仅依赖单一相似度搜索，可能错过相关但表述不同的信息。
    - **优化**：采用**混合检索**（Hybrid Search），结合向量相似度与关键词搜索（BM25）。重排序（Reranking）模型（如`bge-reranker-large`）可对Top-N结果进行精排，显著提升相关性。

4.  **生成阶段提示脆弱**：
    - **坑点**：模型可能忽略检索内容或编造信息。
    - **优化**：在提示中明确指令，如“请严格根据以下上下文回答问题，若无法找到答案则回复‘根据现有资料无法回答’”。使用`stuff`、`map_reduce`等不同文档合并策略处理长上下文。

5.  **数据更新与索引维护**：
    - **坑点**：知识库更新后，向量索引未同步。
    - **优化**：设计增量索引流程，使用支持元数据过滤的向量库（如记录文档时间戳），便于版本管理和增量更新。

RAG的工程化是一个系统性的调优过程，关键在于**持续评估**（使用如`Ragas`、`TruLens`等框架评估忠实度、答案相关性等指标）和**迭代优化**每个环节。理解其核心原理与常见陷阱，是构建可靠RAG应用的基础。

## 💻 代码片段：用Python实现一个简单的语义搜索函数（基于sentence-transformers）

语义搜索通过将文本转换为向量表示，计算语义相似度来实现"理解含义"的搜索，而非简单的关键词匹配。本示例使用 `sentence-transformers` 生成文本嵌入，通过余弦相似度排序返回最相关的文档，支持中英文混合搜索。

```python
"""基于 sentence-transformers 的简易语义搜索"""

import numpy as np
from sentence_transformers import SentenceTransformer
from numpy.typing import NDArray


def semantic_search(
    query: str,
    corpus: list[str],
    top_k: int = 5,
    model: SentenceTransformer | None = None,
) -> list[tuple[int, str, float]]:
    """
    对语料库进行语义搜索，返回与查询最相似的文档。

    Args:
        query: 查询文本
        corpus: 文档列表
        top_k: 返回前 k 个结果，默认 5
        model: 预加载的模型实例，为 None 时自动加载

    Returns:
        [(文档索引, 文档内容, 相似度分数), ...] 按相似度降序

    Example:
        >>> results = semantic_search("什么是AI", ["人工智能是..."])
        >>> for idx, text, score in results:
        ...     print(f"[{score:.4f}] {text}")
    """
    if model is None:
        model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")

    # 生成嵌入向量
    query_vec: NDArray[np.float32] = model.encode(
        [query], normalize_embeddings=True
    )
    corpus_vecs: NDArray[np.float32] = model.encode(
        corpus, normalize_embeddings=True
    )

    # 计算余弦相似度并取 top_k
    similarities: NDArray[np.float32] = corpus_vecs @ query_vec.T
    scores: NDArray[np.float32] = similarities.flatten()
    top_indices: NDArray[np.intp] = np.argsort(scores)[::-1][:top_k]

    return [(int(i), corpus[i], float(scores[i])) for i in top_indices]


# ============ 使用示例 ============
if __name__ == "__main__":
    docs: list[str] = [
        "机器学习是人工智能的核心技术之一",
        "Python 是数据科学领域的主流编程语言",
        "深度学习利用多层神经网络提取特征",
        "自然语言处理让计算机理解人类语言",
        "计算机视觉用于图像识别和目标检测",
        "今天天气真不错，适合出去散步",
    ]

    # 预加载模型（多次搜索时复用，避免重复加载）
    model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")

    query = "人工智能和机器学习有什么关系"
    results = semantic_search(query, docs, top_k=3, model=model)

    print(f"查询：{query}\n" + "-" * 50)
    for rank, (idx, text, score) in enumerate(results, 1):
        print(f"Top{rank} [{score:.4f}] {text}")
```
